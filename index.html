LISTA DE COMANDOS
/play
Toca uma música por URL do YouTube ou nome da faixa (busca automática via YouTube Data API v3).
YouTube URL Busca por nome
/skip
Pula a música atual e passa para a próxima na fila automaticamente. Se não houver próxima, o bot para.
Controle de fila
/previous
Volta para a última música tocada. Funciona mesmo após /stop graças ao estado persistente salvo em state.js.
Histórico
/pause
Pausa a reprodução atual sem sair do canal de voz. O estado da fila é preservado.
Controle
/resume
Retoma a música pausada de onde parou. Só funciona se o bot estiver em estado de pausa.
Controle
/stop
Para a reprodução, limpa a fila e faz o bot sair do canal de voz. A última música fica salva no estado.

+ ADICIONAR AO DISCORD
// OU RODE LOCALMENTE
01
CLONE O REPOSITÓRIO
Faça o clone do projeto e entre na pasta.
git clone https://github.com/enzoka27/discord_bot && cd discord_bot
02
INSTALE AS DEPENDÊNCIAS
Node.js 18+ requerido. Instala discord.js, distube, yt-dlp e demais pacotes.
npm install
03
REGISTRE OS SLASH COMMANDS
Deploya os comandos para o Discord. Só precisa rodar uma vez (ou quando adicionar novos comandos).
node src/deploy-commands.js
04
INICIE O BOT
O bot vai conectar ao Discord e ficar pronto para receber comandos no seu servidor.
node src/index.js
04
SOBRE O PROJETO

Trinity é um bot de música para Discord construído do zero em Node.js, desenvolvido com foco em aprendizado — cada linha de código entendida antes de ser escrita.

O projeto passou por várias evoluções técnicas: de ytdl-core para play-dl e finalmente para DisTube + yt-dlp, a solução estável para autenticação com o YouTube.

Suporte a YouTube (URL e busca via YouTube Data API v3). Arquitetura orientada a eventos com discord.js v14 e carregamento automático de comandos por subpastas.

Node.js discord.js v14 DisTube v5 yt-dlp ffmpeg-static opusscript axios dotenv
